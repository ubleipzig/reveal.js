Probleme bei der Entwicklung

  • abweichende Standard-Konfiguration auf unterschiedlichen Betriebssystemen
  • aufwendige Vorabkonfiguration der beteiligten Services
  • komplexe Einstellungsmöglichkeiten beeinflussen das Ergebnis
  • Ressourcen unterschiedlicher Entwicklungsumgebungen überschneiden sich

an VuFind beteiligte Services

  • PHP
  • Apache2
  • MySQL
  • Solr
  • PHP ist der Applikationsserver:
    • führt VuFind aus und prozessiert Benutzereingaben
    • ist das Bindeglied aller beteiligter Services
  • Apache2 ist der Webserver
    • Schnittstelle zwischen Benutzer und VuFind
    • nimmt Benutzereingaben entgegen
    • gibt Benutzereingaben an VuFind
    • übermittelt statische Inhalte wie Theme und Layout an den Benutzer
    • übermittelt dynamische Inhalte, prozessiert von VuFind an den Benutzer
  • MySQL ist der Datanbankserver
    • benutzerrelevante Inhalte
    • VuFind-spezifische Laufzeitinformationen, wie Sessiondaten
  • Solr ist der Indexserver
    • stellt die eigentlichen Katalog-Informationen zur Verfügung
    • keine Docker-Komponente, da Aufbau zu komplex

Docker Compose

  • beschreibt Services
  • Apache2
  • MySQL
  • Volumes
